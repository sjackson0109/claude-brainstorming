Prerequisites

Before installing the Brainstorming Skill, ensure you have the following tools and accounts set up.

Claude Code

The Claude Code CLI must be installed and accessible from your terminal.

Git

Git must be installed and available on your system PATH.

Active Subscription

An active Claude subscription (Pro/Team/Enterprise) or a valid API key.

Verify Your Environment

Open your terminal and run the following commands to confirm everything is in place:

# Confirm Claude Code is installed
claude --version

# Confirm Git is available
git --version

If claude is not recognised, follow the official Claude Code installation guide first.

Installation Methods

Choose the method that best suits your workflow. The marketplace approach is recommended for most users.

Recommended. The fastest way to install — two commands in your Claude Code session.

1

Add the marketplace source

Register the Brainstorming Skill marketplace within your Claude Code environment:

/plugin marketplace add sjackson0109/claude-brainstorming
2

Verify the marketplace listing

Confirm that the marketplace has been added successfully:

/plugin marketplace list

You should see brainstorming-skills in the output.

3

Install the skill

Install the Brainstorming Skill from the registered marketplace:

/plugin install brainstorming@brainstorming-skills

That’s it! The skill is now available in your Claude Code sessions. Jump to Verifying Installation to confirm.

If you prefer to manage the skill files yourself, you can clone the repository and import the skill manually.

1

Clone the repository

Clone the Brainstorming Skill repository to your local machine:

git clone https://github.com/sjackson0109/claude-brainstorming.git
cd claude-brainstorming
2

Open Claude Code settings

Within your Claude Code session, navigate to Settings → Skills.

3

Import the skill file

Import the brainstorming.skill file from the cloned repository. This is the packaged skill archive containing all 150 domain reference files.

Alternatively, you can reference the brainstorming/ directory directly if you prefer working with the raw skill files.

4

Confirm activation

The skill should now appear in your Skills list. Verify by checking the active skills in your settings panel.

For team environments, you can pre-configure the skill so it is automatically available to all team members via shared project settings.

1

Locate your project settings

Find or create the .claude/settings.json file in the root of your project repository.

2

Add the marketplace and plugin configuration

Add the following configuration to your .claude/settings.json:

settings.json
{
  "extraKnownMarketplaces": {
    "brainstorming-skills": {
      "source": {
        "source": "github",
        "repo": "sjackson0109/claude-brainstorming"
      }
    }
  },
  "enabledPlugins": {
    "brainstorming@brainstorming-skills": true
  }
}
3

Commit and push

Commit the .claude/settings.json file to your repository. When team members open their Claude Code sessions, the skill will be automatically installed and available.

git add .claude/settings.json
git commit -m "Add Brainstorming Skill auto-install config"
git push

All team members will now automatically have the Brainstorming Skill available when they start a Claude Code session in this project.

Verifying Installation

After installing the skill, confirm that it is active and ready to use.

1

Check installed plugins

List your installed plugins to confirm the Brainstorming Skill is present:

/plugin list

You should see brainstorming@brainstorming-skills listed with an active status.

2

Run a quick test

Try a simple brainstorming prompt to confirm everything is working:

Test Prompt

“Use your Brainstorming skill to help me brainstorm ideas for improving team productivity.”

Expected Behaviour

Claude should acknowledge the Brainstorming Skill, identify the relevant domains (such as productivity and organisational development), and provide structured, expert-level suggestions.

If Claude responds with domain-specific brainstorming output, the skill is correctly installed and working.

Using the Skill

The Brainstorming Skill works by analysing your prompt, detecting the relevant domains, and applying specialist expertise from its library of 150 reference skills. Here’s how to get the most out of it.

Basic Prompt Patterns

Start your prompt with a clear reference to the Brainstorming Skill. The following patterns all work:

# Direct skill invocation
"Use your Brainstorming skill to help me with..."

# Natural language trigger
"Brainstorm ideas for..."

# Domain-specific request
"Help me brainstorm a cybersecurity strategy for..."

Single-Domain Requests

When your request targets a single area of expertise, the skill routes to the most relevant domain and applies its specialised frameworks.

Single-Domain Prompt

“Use your Brainstorming skill to help me design a comprehensive API authentication strategy for a multi-tenant SaaS platform.”

Skill Activation

Routes to API design expertise, covering OAuth 2.0 flows, JWT management, tenant isolation patterns, and rate limiting strategies.

api-design.md
Single-Domain Prompt

“Brainstorm approaches for implementing a green supply chain initiative in our manufacturing business.”

Skill Activation

Routes to sustainability expertise, applying circular economy frameworks, emissions tracking, and supplier assessment methodologies.

sustainability-metrics.md

Multi-Domain Requests

The skill truly shines when your challenge spans multiple areas. It automatically detects and combines relevant domains for a comprehensive response.

Multi-Domain Prompt

“Help me brainstorm a digital transformation strategy that includes cybersecurity considerations, change management for staff, and a pricing model for the new digital services.”

Skill Activation

Combines expertise from four domains to deliver a holistic strategy covering technology transformation, security architecture, organisational change psychology, and revenue modelling.

digital-strategy.md cybersecurity.md change-management-psychology.md advanced-pricing-strategy.md

Advanced Techniques

Iterative Refinement

Start with a broad brainstorming request and then refine. The skill maintains context across your conversation:

# Step 1: Initial broad brainstorm
"Use your Brainstorming skill to brainstorm strategies for entering the European market."

# Step 2: Narrow the focus
"Now focus specifically on the regulatory and compliance challenges."

# Step 3: Add another domain
"Also consider the pricing strategy implications for each market."

Constraint-Based Brainstorming

Provide constraints to receive more targeted, actionable output:

"Use your Brainstorming skill to brainstorm cost-effective cybersecurity
 improvements for a startup with fewer than 20 employees and a limited
 budget. Prioritise quick wins that can be implemented within 30 days."

Comparative Analysis

Ask the skill to compare approaches across frameworks:

"Brainstorm the pros and cons of adopting a microservices architecture
 versus a modular monolith, considering our team of 8 developers and
 current deployment pipeline."

Tip: The more context you provide about your situation, constraints, and goals, the more targeted and valuable the brainstorming output will be.

Keeping Updated

The Brainstorming Skill is actively maintained. Keep your installation up to date to benefit from new domain skills and improvements.

1

Update the marketplace

Refresh the marketplace to fetch the latest version of the skill catalogue:

/plugin marketplace update brainstorming-skills
2

Update the installed plugin

Apply the latest version to your local installation:

/plugin update brainstorming@brainstorming-skills

Tip: Check for updates periodically, especially after new releases are announced on the GitHub Releases page.

Uninstalling

If you need to remove the Brainstorming Skill, follow these two steps:

1

Uninstall the plugin

Remove the installed plugin from your Claude Code environment:

/plugin uninstall brainstorming@brainstorming-skills
2

Remove the marketplace source

Optionally, remove the marketplace source if you no longer need it:

/plugin marketplace remove brainstorming-skills

If you used the Team Setup method, remember to also remove the configuration from your project’s .claude/settings.json file.

Troubleshooting

Encountering issues? Here are solutions to common problems.

This usually means the marketplace source hasn’t been added yet. Run the following command first:

/plugin marketplace add sjackson0109/claude-brainstorming

Then retry the install command. Also verify your internet connection and that you can reach GitHub.

Ensure you’re explicitly mentioning brainstorming in your prompt. Try phrases like:

  • “Use your Brainstorming skill to…”
  • “Brainstorm ideas for…”
  • “Help me brainstorm…”

Also confirm the plugin is active by running /plugin list and checking that it shows as enabled.

Verify the file is located at exactly .claude/settings.json in the repository root. Ensure the JSON is valid (no trailing commas, correct bracket pairing). Team members need to pull the latest changes before the configuration takes effect.

Ensure you have an active Claude subscription or valid API key configured. Check your Claude Code authentication status:

claude --version

If you’re behind a corporate firewall or proxy, ensure that both github.com and the Anthropic API endpoints are accessible.

Open an issue on the GitHub Issues page. Include details about your environment, the steps to reproduce, and any relevant error messages. Feature requests for new domains are also welcome.
