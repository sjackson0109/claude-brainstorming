# Mobile Application Penetration Testing Brainstorming Skill

## Purpose

The Mobile Application Penetration Testing Brainstorming Skill specialises in comprehensive offensive security assessment strategy for mobile applications across Android and iOS platforms. This skill empowers strategic thinking through systematic mobile vulnerability discovery, platform-specific attack vector analysis, and comprehensive security testing methodologies for mobile application ecosystems.

## Core Capabilities

### Mobile Security Assessment Framework
- **Cross-Platform Testing Strategy**: Comprehensive security assessment approach covering both Android and iOS applications with platform-specific vulnerability analysis
- **Mobile API Security Testing**: Strategic assessment of mobile application backend APIs, authentication flows, and data transmission security
- **Runtime Application Security Testing**: Dynamic analysis of mobile applications during execution including memory manipulation and runtime behaviour modification

### Platform-Specific Security Analysis
- **Android Application Security**: Comprehensive APK analysis, Android manifest security review, and Android-specific vulnerability assessment including Intent hijacking and storage security
- **iOS Application Security**: Systematic IPA analysis, iOS keychain security testing, and iOS-specific vulnerability assessment including jailbreak detection bypass
- **Mobile Device Security Integration**: Assessment of mobile device security features integration including biometric authentication and secure enclave utilisation

### Mobile Application Ecosystem Testing
- **Mobile Backend Security**: Comprehensive testing of mobile application server-side components and cloud service integrations
- **Third-Party Library Security**: Strategic analysis of mobile application dependencies and third-party SDK security implementations
- **Mobile DevOps Security**: Assessment of mobile application build pipelines, code signing, and distribution security

## Implementation Framework

### Phase 1: Mobile Application Reconnaissance and Analysis (65 minutes)
- Conduct comprehensive mobile application reverse engineering and static analysis to understand application architecture and functionality
- Perform mobile application fingerprinting including technology stack identification, framework analysis, and third-party integration mapping
- Develop mobile-specific attack surface analysis including API endpoints, local storage, and inter-app communication channels

### Phase 2: Runtime Security Testing and Exploitation (85 minutes)
- Execute dynamic mobile application testing using instrumentation frameworks and runtime manipulation techniques
- Perform comprehensive mobile API testing including authentication bypass, parameter manipulation, and business logic vulnerabilities
- Conduct mobile-specific vulnerability testing including SSL pinning bypass, certificate validation bypass, and cryptographic implementation analysis

### Phase 3: Platform Security and Data Protection Testing (70 minutes)
- Analyse mobile application data storage security including local databases, shared preferences, and keychain implementations
- Test mobile application inter-process communication security including Intent handling, URL schemes, and background processing
- Assess mobile application privacy and data protection compliance including permission models and data leakage prevention
