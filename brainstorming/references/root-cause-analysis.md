# Root Cause Analysis Brainstorming Skill

## Purpose

The Root Cause Analysis Brainstorming Skill specialises in systematic investigation and identification of fundamental causes underlying problems, failures, and undesirable events. This skill empowers strategic thinking through investigative methodologies, causal analysis frameworks, and the creation of comprehensive solutions that address core issues rather than symptoms, ensuring sustainable problem resolution and prevention of recurrence.

## Core Capabilities

### Systematic Problem Investigation and Analysis
- **Problem Definition and Scoping**: Clearly define problems including problem statements, boundary setting, and stakeholder impact assessment
- **Evidence Collection and Data Gathering**: Systematic evidence gathering including data collection, witness interviews, and documentation review
- **Timeline and Sequence Analysis**: Reconstruct event sequences including chronological analysis, dependency mapping, and critical path identification

### Causal Analysis Methodologies and Techniques
- **5 Whys and Iterative Questioning**: Implement iterative questioning techniques including progressive why analysis and logical chain development
- **Fishbone Diagram and Ishikawa Analysis**: Create cause-and-effect diagrams including systematic category analysis and comprehensive cause identification
- **Fault Tree Analysis and Logic Models**: Develop logic-based analysis including fault tree construction and probabilistic failure analysis

### Solution Development and Implementation
- **Corrective Action Planning**: Design comprehensive corrective actions including immediate fixes, long-term solutions, and systemic improvements
- **Preventive Measures and Controls**: Implement preventive controls including process changes, system improvements, and risk mitigation strategies
- **Solution Validation and Effectiveness**: Validate solution effectiveness including implementation monitoring, outcome assessment, and recurrence prevention

## Implementation Framework

### Root Cause Analysis Investigation Process

**Phase 1: Problem Definition and Investigation Planning (90 minutes)**
- Conduct problem clarification and definition including problem statement development, scope boundary setting, and impact assessment
- Develop investigation strategy including methodology selection, resource allocation, and investigation timeline
- Establish investigation team including role assignments, expertise requirements, and stakeholder engagement
- Create investigation framework including data collection protocols, analysis procedures, and reporting standards

**Phase 2: Evidence Collection and Causal Analysis (120 minutes)**
- Implement comprehensive data collection including quantitative data gathering, qualitative information collection, and physical evidence examination
- Conduct systematic causal analysis including multiple methodology application, hypothesis development, and cause verification
- Perform stakeholder interviews including witness statements, expert consultations, and subject matter expert engagement
- Create comprehensive cause mapping including direct causes, contributing factors, and underlying systemic issues

**Phase 3: Solution Development and Implementation Planning (90 minutes)**
- Design multi-layered solutions including immediate corrective actions, intermediate improvements, and long-term systemic changes
- Develop implementation strategies including action plans, resource requirements, and timeline development
- Create monitoring and validation systems including effectiveness measurement, progress tracking, and success criteria
- Establish continuous improvement processes including lessons learned capture, knowledge sharing, and prevention enhancement
