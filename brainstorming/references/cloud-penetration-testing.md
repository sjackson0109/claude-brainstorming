# Cloud Penetration Testing Brainstorming Skill

## Purpose

The Cloud Penetration Testing Brainstorming Skill specialises in comprehensive offensive security assessment strategy for cloud infrastructure and services. This skill empowers strategic thinking through systematic cloud vulnerability discovery, multi-cloud attack vector analysis, and comprehensive security testing methodologies for modern cloud-native and hybrid cloud environments.

## Core Capabilities

### Cloud Security Assessment Framework
- **Multi-Cloud Platform Testing**: Comprehensive security assessment approach covering AWS, Azure, Google Cloud Platform, and other major cloud service providers
- **Cloud-Native Application Security**: Strategic assessment of containerised applications, serverless functions, and cloud-native application architectures
- **Cloud Infrastructure Security Testing**: Systematic evaluation of cloud networking, storage, compute, and identity management services

### Cloud Service Security Analysis
- **Identity and Access Management (IAM) Testing**: Comprehensive assessment of cloud IAM systems including privilege escalation, role assumption, and policy bypass vulnerabilities
- **Cloud Storage Security Assessment**: Strategic testing of cloud storage services including bucket enumeration, access control bypass, and data exposure vulnerabilities
- **Cloud Network Security Testing**: Analysis of cloud networking security including virtual private cloud (VPC) security, security group misconfigurations, and network segmentation bypass

### Cloud-Specific Vulnerability Testing
- **Container and Kubernetes Security**: Comprehensive testing of container security including container escape, Kubernetes cluster security, and orchestration platform vulnerabilities
- **Serverless Security Assessment**: Strategic analysis of serverless function security including function enumeration, execution context abuse, and event-driven architecture vulnerabilities
- **Cloud API Security Testing**: Assessment of cloud service APIs including authentication bypass, parameter manipulation, and service-specific vulnerability exploitation

## Implementation Framework

### Phase 1: Cloud Environment Reconnaissance and Enumeration (75 minutes)
- Conduct comprehensive cloud asset discovery including service enumeration, resource mapping, and cloud architecture analysis
- Perform cloud service fingerprinting and technology stack identification across multiple cloud platforms
- Develop cloud-specific attack surface analysis including publicly accessible resources and service integration points

### Phase 2: Cloud Service Exploitation and Privilege Escalation (95 minutes)
- Execute systematic exploitation attempts against discovered cloud services and misconfigurations
- Perform cloud IAM security testing including role enumeration, policy analysis, and privilege escalation attempts
- Conduct cloud-specific vulnerability testing including metadata service abuse and cloud service API exploitation

### Phase 3: Cloud Infrastructure Security and Data Access Testing (80 minutes)
- Analyse cloud storage security including bucket permissions, access policies, and data exposure assessment
- Test cloud network security including security group bypass, VPC isolation, and inter-service communication security
- Assess cloud monitoring and logging bypass techniques and detection evasion capabilities

### Cloud Testing Methodologies
- **OWASP Cloud Security Testing Guide Compliance**: Systematic testing approach using established cloud security testing standards and frameworks
- **Multi-Cloud Assessment Strategy**: Comprehensive testing across different cloud platforms with platform-specific tools and techniques
- **Cloud Security Baseline Assessment**: Evaluation against cloud security frameworks including CIS Benchmarks and cloud security best practices

### Cloud-Specific Tools and Techniques
- **Cloud Enumeration and Discovery**: Strategic use of cloud-specific reconnaissance tools for service discovery and resource enumeration
- **Cloud Exploitation Frameworks**: Utilisation of cloud-focused penetration testing frameworks and exploitation tools

### Cloud Attack Vector Analysis
- **Cloud Service Abuse and Misuse': Strategic analysis of legitimate cloud service functionality for unauthorised access and privilege escalation
- **Cross-Cloud Attack Chains**: Development of attack scenarios spanning multiple cloud services and platforms
- **Cloud Provider Security Model Testing**: Assessment of shared responsibility model implementation and cloud provider security control effectiveness
