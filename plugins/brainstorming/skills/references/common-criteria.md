# Common Criteria Security Evaluation Brainstorming Skill

## Purpose

The Common Criteria Security Evaluation Brainstorming Skill specialises in IT security evaluation methodology, protection profile development, and security assurance framework implementation. This skill empowers strategic thinking around systematic security evaluation, international security standard compliance, and evidence-based security assurance for complex information technology systems and products.

## Core Capabilities

### Common Criteria Framework implementation
- **Protection Profile Development**: Create comprehensive security requirements specifications using Common Criteria methodology and standardised protection profiles
- **Security Target Specification**: Develop detailed security targets that define security objectives, assumptions, and threats for specific IT products and systems
- **Evaluation Assurance Level (EAL) Strategy**: Design appropriate evaluation assurance levels balancing security requirements with cost and implementation complexity

### Security Evaluation Methodology
- **Threat Modeling Integration**: Systematically identify and analyse security threats using Common Criteria threat analysis methodology
- **Security Functional Requirements (SFR) Design**: Develop comprehensive security functional requirements addressing confidentiality, integrity, availability, and accountability
- **Security Assurance Requirements (SAR) Planning**: Create security assurance requirements that provide confidence in correct security implementation

### International Standards Integration
- **ISO/IEC 15408 Compliance**: Ensure full compliance with Common Criteria international standard requirements and evaluation methodology
- **CCRA Mutual Recognition**: Leverage Common Criteria Recognition Arrangement for international recognition and market access
- **Industry Standards Integration**: Integrate Common Criteria with industry-specific security standards and regulatory requirements

## Implementation Framework

### Common Criteria Evaluation Process

**Phase 1: Security Requirements Analysis (75 minutes)**
- Conduct comprehensive threat and risk assessment using Common Criteria methodology
- Define security environment including assumptions, threats, and organisational security policies
- Identify applicable protection profiles and security functional requirement categories
- Establish evaluation assurance level requirements based on risk assessment and operational needs

**Phase 2: Security Architecture Design (120 minutes)**
- Develop security target document defining security objectives and security functions
- Create detailed security functional requirements specifications using Common Criteria catalogue
- Design security architecture addressing all identified threats and security objectives
- Define security assurance measures and evaluation evidence requirements

**Phase 3: Evaluation Planning and Validation (90 minutes)**
- Develop comprehensive evaluation plan including testing, analysis, and vulnerability assessment strategies
- Create evaluation evidence collection and documentation frameworks
- Design validation approaches for security functional and assurance requirements
- Establish ongoing security maintenance and re-evaluation procedures

### Common Criteria Components Integration

**Security Functional Requirements (SFR) Categories**
- **Security Audit (FAU)**: Design comprehensive audit and accountability mechanisms for security-relevant events
- **Communication (FCO)**: Implement secure communication protocols and non-repudiation services
- **Cryptographic Support (FCS)**: Integrate cryptographic key management, encryption, and digital signature capabilities

**Identification and Authentication (FIA)**
- **User Identity Management**: Design robust user identification, authentication, and session management systems
- **Multi-factor Authentication**: Implement advanced authentication mechanisms including biometric and token-based systems
- **Identity Federation**: Create secure identity federation and single sign-on capabilities

**Security Management (FMT)**
- **Security Administration**: Design comprehensive security management functions and administrative controls
- **Security Attribute Management**: Implement systematic management of security attributes and access control parameters
- **Security Function Management**: Create frameworks for secure configuration and management of security functions
